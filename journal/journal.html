Week 2

I thought this weeks beginner tut was an excellent way to get into HTML (and Git). The tutorial work was fairly simple to get through with clear instructions and had a diverse range of tasks to complete. Although, some tasks towards the end were a bit more fiddly to work with. I had to go look up how to do some of the tasks like how do I make all the form elements mandatory and password specifications? I didn't need to use ChatGPT to find out as there were plenty of sources to help me discover the answer and I was able to make sense of how they worked, especially considering most of this is still basic stuff. Maybe I wasn't supposed to do them just yet, but I still did them anyway because I like to do everything that's there. I also liked how we were given examples of what our finished page should look like, as it was another source of trying to match it based on your current knowledge without needing to refer to the activity manual (asides from the tasks with specifications).
It was a little challenging trying to set up with Gitea with ictteach. I am familar with Git already, so I was ready to create a new repository on GitHub, but I do like how most things are already set up for us to avoid potential issues, which I know many who are inexperienced will make. I ended up losing all my finished progess on `index.html` because of a failed save, which was a pain to recover. Hopefully the team stuff goes all well without too many (if any) hiccups. Having had some disasterous experiences with version control in the past, I hope to avoid repeating those occurances at all costs.

Week 3

This week's tutorial was quite fun in "dressing up" the page to make it look nice. It was a little confusing having to read instructions from different sources (MyLO and the .css file) while simultaneously having to update the .html files to accommodate the CSS changes. I feel like it was better if I read through the whole MyLO page instructions before I started working on the .css file. There is also a lot of things to read over in the .css file, which is all fine but the structure of some comments change which throws me in for a loop because it's easy to gloss over instructions. The registration form was the main one as it threw me off because I completely missed the part where it told me display the form as a grid layout. I got through it all in the end though.
One thing I did learn this week is to be careful when using commas. I knew to be careful of using spaces in CSS, but I was confused on why margins and paddings were not displaying as they should, until I realised that the commas were causing the rest of numbers to be ignored. It's a little confusing because there are some parts that require commas, but at least I now know the importance of not using commas for the important styles.

Week 5

This week's tutorial work with JavaScript was quite simple and straightforward. I felt really in the zone with how similar it is to other languages such as C, Java, etc. So, I didn't really learn much because I was already confident with most of the work here. It was a little tricky getting my head around the concept document.getElementById("id").innerHTML but once I saw it in action, things started to click.
Seeing how JavaScript works with HTML now though, I have some ideas in trying to incorporate it such as making a default header/footer style rather than having to copy and paste huge chunks of code.

Week 6

This week's tutorial was quite complex to follow. There was a whole lot of things thrown onto us and it felt like we were just given the goal without being told how to get there. I first stated out by doing simple if statements, but I realised that the code became quite restricted in its functionality, unable to remove the error messages if the user updated the input correctly. I walked back into Week 5 and decided to create a massive array (checks), which provided great flexibility in what I could do. Getting my head around it was a little tricky because of all the requirements, so I'm not sure if that was what I was supposed to do or not, but if it was, a guide going through how to build it would've been helpful. From there though, the rest was much easier with the foundation laid out. I wanted to try and push myself a little more to make the form more dynamic by making the form grid change from 2 to 3 columns and vice versa. I had to rearrange some functions to ensure that this could work properly and not screw up the whole form format. But I got there in the end.
Learning how to use querySelector() makes calling for classes, ids, and names so much easier. Being able to use it, but more specifically classList and style to add, remove, and modify CSS styling opens things up in a dynamic sense.
I also figured out that you could nest "ternary operators" within each other, making for some interesting and complex if/else statements. It does get messy to read though, so I needed to add parentheses to help make it more readable.
